Add stdin input support and a text-adventure demo. The compiled OCaml currently has output-only I/O (print_int, print_endline, putc) but no way to read from the user — so every interactive demo is impossible. Goal: implement the smallest viable input stack (getc -> read_line) in the interpreter runtime, then build an idiomatic OCaml text adventure (rooms as ADTs, commands parsed by pattern matching) that demonstrates why OCaml is a good language for this class of program. Steps will: (1) add getc primitive wired to UART, (2) add read_line built on getc, (3) smoke-test interactive I/O with echo + guess-the-number, (4) build the text-adventure demo with 3-4 rooms, basic commands (n/s/e/w/look/inventory/quit), and variant-typed world state.
